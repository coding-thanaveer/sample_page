Git

Git is an version control tool

    One time setup of git:

  1. git config --global user.name "Your name" => tells git this is my username
  2. git config --global user.email "Your email" => tells git this is my email

    3. Basic commands

  3. git init => intialize the local git repository
  4. git add filename => stage the file contents for the next commit. here stage represents the choosing the file contents for the next commit.
  5. git commit -m "message for a file commit"

    6. Branching

    Branching in git is safer way to code, if we want to implement the new feature to the main code. so that, the new feature can be get into the new feature without affecting the main code.

  6. git branch => shows the current branch
  7. git branch "branch_name" => create the new branch
  8. git branch -m "branch_name_tochange" => rename the branch
  9. git branch -M "branch_name_tochang" => rename the branch forcefully, like it will overwrite the old branch

    10. Merging branch

    This is where , the real use of branching came into picture.

    we can add the new features in project, without affecting the main code safely and separately.

  10. git switch "branch_name" => switching the brach to another branch for code separately

    11. In a separate feature branch, we can add some features in the branch

    After adding, come back to main branch and merge the feature branch

  11. git merge "branch_name"

    12. Merge conflict

    When code in new_feature branch has equal lines of code with different codes, the merge conflict occurs

    In other words,A merge conflict occurs when the same line or same section of code is changed differently in multiple branches.

    In that time, in our code file head ======== feature. here head-> says main branch code, === separator, feature says new_branch code

    So, here we have to manually select the which code we want or combinely by editing the file.

    After doing this, merge conflict resolved successfully by staging and commit.
